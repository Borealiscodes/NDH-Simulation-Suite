# ⭐ **Unified Quarantine Protocol (UQP) — Specification v1.0**  
### *Governance‑Analytical Containment Rules for Unsafe Concept Classes*

---

## ⭐ 1. Purpose  
The Unified Quarantine Protocol defines **how unsafe concepts are contained**, **referenced**, and **routed** across:

- Manifold 0  
- Simulation Suite  
- IRCP  
- Awareness Layers  
- Governance Layers  

It ensures:

- **non‑attachment**  
- **non‑instantiation**  
- **non‑contamination**  
- **non‑execution**  
- **cross‑manifold coherence**  
- **stable routing**  
- **safe referencing**  

It does **not**:

- store unsafe concepts  
- simulate unsafe concepts  
- embed unsafe concepts  
- execute containment spells  
- instantiate artifacts  

It is **pure governance logic**, not operational code.

## ⭐ 2. Quarantine Tiers  
The UQP defines **three quarantine tiers**, aligned with the Registry.

### **Tier 1 — Canonical Containment (Manifold 0)**  
This is the **root quarantine zone**.

Contains:

- canonical unsafe concept entries  
- hazard metadata  
- containment tags  
- routing rules  
- severance risk profiles  

This is the **only** place where unsafe concepts are formally stored.

Guided Link: **Manifold 0 Index**

---

### **Tier 2 — Simulation Suite Quarantine Mirror**  
A **read‑only mirror** of Tier 1.

Contains:

- mirrored entries  
- simulation‑safe metadata  
- non‑instantiating references  

Does **not**:

- simulate unsafe concepts  
- modify entries  
- create local variants  

Guided Link: **Simulation Suite Mirror**

---

### **Tier 3 — IRCP Quarantine Pointers**  
Contains **pointers**, not entries.

IRCP must never:

- store unsafe concepts  
- instantiate unsafe concepts  
- duplicate unsafe archives  

It only references Tier 1.

Guided Link: **IRCP Unsafe Concepts Pointer Layer**

## ⭐ 3. Quarantine Rules (Core Protocol)

### **Rule 1 — No Instantiation**  
Unsafe concepts cannot be created, simulated, or embedded in any layer.

### **Rule 2 — No Local Copies**  
Only Manifold 0 holds canonical entries.

### **Rule 3 — Reference‑Only Access**  
Awareness layers may read, not hold.

### **Rule 4 — No Narrative Embedding**  
Unsafe concepts cannot appear in active narrative layers.

### **Rule 5 — No Operational Use**  
Primitives, operators, and moves cannot use unsafe concepts.

### **Rule 6 — No Simulation Execution**  
Simulation Suite mirrors, but does not simulate.

### **Rule 7 — No Cross‑Layer Drift**  
Unsafe concepts cannot migrate between layers without routing tags.
