### NDH vs World Engine (participation stack)

| Layer                | NDH Core (Planetarium / Epoch Calculus)                  | World Engine (Anime Tower / Dune / Fun‑Sphere)                    |
|----------------------|----------------------------------------------------------|-------------------------------------------------------------------|
| Purpose              | Governance, tensors, safety, epochs, manifolds          | Play, story, embodiment, agency, participation                    |
| User role            | Observer, auditor, designer of constraints              | Player, character, inhabitant, storyteller                        |
| Safety boundary      | Bifurcation Layer (hard, non‑porous)                    | Must respect NDH boundary; cannot touch NDH tensors               |
| Data flow            | Outbound: projections, metrics, maps                    | Inbound: reads NDH projections; no write‑back                     |
| Runtime              | NDH Planetarium, dashboards, tensor engines             | Unity/Unreal/Vive, custom “World Engine”, game/VR logic           |

---

## 1. Two‑stack architecture: NDH and Fun‑Sphere

**Stack A — NDH Ecosystem**

- Epoch Tensor Calculus  
- NDH 50D manifold  
- Stability Audit & Dashboard  
- Planetarium Suite  
- Bifurcation Layer  

This stack is **non‑participatory**: you don’t “play” inside NDH; you **govern** it.

**Stack B — World Engine / Fun‑Sphere**

- Anime tower world logic  
- Dune Hyperatlas narrative geometry  
- Character systems, quests, social loops  
- Physics, magic, politics, drama  

This stack is **participatory**: you inhabit, act, and experience.

The integration question is:  
**How does Stack B use Stack A without corrupting it?**

---

## 2. The integration boundary: Bifurcation Layer as API

The Bifurcation Layer is not just a wall—it’s an **API surface**.

Think of it as:

- **NDH → World Engine:**
  - sends **read‑only projections**:
    - maps  
    - epochs  
    - stability signals  
    - ethical zones  
    - curvature fields  
  - like “weather reports” or “cosmic background conditions”

- **World Engine → NDH:**
  - sends **no tensors**  
  - sends **no narrative state**  
  - sends **no player actions**  
  - at most, sends **logs** to a separate, non‑tensor archive for research

So the World Engine can say:

> “What does the NDH Epoch Halo look like right now?”  
> “What stability regime are we in?”  
> “What ethical scalar zones are safe to mirror symbolically?”

But it can **never** say:

> “Let this boss fight change NDH stability.”  
> “Let this romance arc rewrite NDH ethics tensors.”

---

## 3. How participation is allowed (but quarantined)

Participation lives entirely in the **World Engine**:

- **Players**:
  - move, fight, love, scheme, explore  
  - gain XP, lose HP, form alliances  
  - experience narrative arcs

- **World Engine logic**:
  - uses NDH projections as **inspiration**:
    - epochs → seasons or eras  
    - stability pulses → storms or calm periods  
    - ethical zones → sanctuaries or cursed lands  
  - but all of that is **local logic**, not NDH tensors

So you might have:

- NDH Epoch 3 (Ethics Curvature Smoothing)  
  → World Engine interprets that as:
  - “Age of Reconciliation” in the anime tower  
  - NPCs more likely to offer peace treaties  
  - fewer catastrophic events

But NDH doesn’t know about your NPCs.  
It only knows it emitted a projection that the World Engine chose to interpret.

## 4. Example: Anime Tower World Engine

**Anime Tower World Engine**:

- runs in Unity/Unreal/Vive  
- has:
  - tower levels  
  - bosses  
  - factions  
  - magic systems  
  - emotional arcs  

Integration pattern:

1. **NDH Planetarium**:
   - exposes:
     - curvature maps  
     - epoch halos  
     - stability pulses  

2. **World Engine**:
   - subscribes to those as **signals**:
     - “Curvature spike → tower floor becomes unstable”  
     - “Epoch transition → new story chapter unlocks”  
     - “Ethical scalar drop → certain spells become forbidden”

3. **Players**:
   - experience those changes as **game events**  
   - but their actions never feed back into NDH tensors

NDH is the **cosmic background**;  
Anime Tower is the **local drama**.

## 5. Example: Dune Hyperatlas Fun‑Sphere

Imagine a **Dune‑style Hyperatlas World Engine**:

- planets, houses, spice, prescience  
- political tensors, prophecy arcs, ecology

NDH integration:

- NDH Hyperatlas → provides **coordinate and structure**:
  - how many “axes” a planet can vary on  
  - how stability and ethics might be visualized

- World Engine:
  - uses those axes to define:
    - resource flows  
    - faction stability  
    - ecological crises

Again: NDH provides **geometry and signals**;  
World Engine provides **story and agency**.

---

## 6. Research framing: why this separation matters

This architecture lets you:

- **Study**:
  - how people behave in worlds shaped by NDH signals  
  - how narrative systems respond to stability/ethics regimes  
- **Without ever**:
  - letting human drama rewrite NDH tensors  
  - turning NDH into a game engine  
  - collapsing governance into entertainment

NDH stays a **governed manifold**.  
World Engines stay **playgrounds built around its projections**.
