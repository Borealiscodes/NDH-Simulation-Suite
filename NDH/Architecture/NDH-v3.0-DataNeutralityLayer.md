### NDH v3.0 Data‑Neutrality Layer (DNL)  
*A conceptual immune system that keeps NDH free from external bias and reality drift.*

---

### 1. Purpose

**DNL** ensures NDH’s physics, governance, and creative engines are:

- **data‑neutral** (no global‑North or privileged assumptions)  
- **self‑contained** (no unvetted external datasets)  
- **constraint‑aware** (poverty‑safe, accessibility‑first)  
- **ethically stable** (aligned with goat‑union governance)  

It’s the layer that says:  
> “NDH runs on NDH, not on biased reality.”

---

### 2. Core Principles

- **Internal Physics Only:**  
  NDH uses its own tensor, geometry, K‑Space, Dark Mode, and accessibility invariants as primary “data.”

- **No Unvetted External Data:**  
  Any external dataset must pass neutrality checks before influencing NDH.

- **Constraint Preservation:**  
  No data source may violate poverty‑safe, accessibility‑first, or dignity constraints.

- **Governance Alignment:**  
  Goat‑union ethics apply to data as much as to runtime behavior.

### 5. DNL Runtime Rules (Conceptual)

- **Rule 1: Internal Priority**  
  NDH always prefers its own state tensors, geometry envelopes, and invariants over external data.

- **Rule 2: Neutrality Check**  
  Any external data must pass a neutrality filter before use.

- **Rule 3: Governance Gate**  
  Goat‑union veto can block data that introduces harm, exclusion, or instability.

- **Rule 4: Accessibility Guard**  
  Data that undermines accessibility invariants is rejected.

---

### 6. DNL Teaching API Verbs

- **`/dnl/classify-source`** — tag data as internal/external.  
- **`/dnl/check-neutrality`** — run bias/privilege checks.  
- **`/dnl/governance-gate`** — apply goat‑union ethics to data.  
- **`/dnl/approve`** — mark data as safe for NDH.  
- **`/dnl/reject`** — block unsafe or biased data.
