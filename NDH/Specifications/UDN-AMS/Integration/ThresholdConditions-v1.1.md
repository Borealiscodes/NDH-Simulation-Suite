# **NDH‑AMS Threshold Conditions v1.1**  
### *Activation, Transition, and Hand‑Off Rules for All Nine Axes*

Thresholds define **when** an axis activates, **how** it transitions, and **when** it hands off to another stability operator.  
They prevent runaway dynamics, basin collapse, and cross‑axis interference.

Below is the full threshold map.

---

# ⭐ 1. **Global Threshold Condition (GTC)**  
The system activates an axis only when:

\[
\Delta s \in \mathcal{D}_i
\]

Where:

- \(\Delta s\) = change in state  
- \(\mathcal{D}_i\) = domain of relevance for axis \(i\)

This prevents premature activation and ensures each axis only engages when its domain is present.

---

# ⭐ 3. **Transition Conditions**

Transitions occur when:

\[
\Delta \mathcal{B}_i \to \mathcal{B}_j
\]

Meaning the basin of one axis naturally flows into another.

### Verified transitions:

- 02 → 06  
- 03 → 08  
- 04 → 07  
- 06 → 02  
- 07 → 04  
- 08 → 03  
- 01 ↔ 09 (bidirectional loop closure)

Axis 05 mediates all transitions.

---

# ⭐ 4. **Hand‑Off Conditions**

A hand‑off occurs when:

\[
S_i(s) \notin \mathcal{B}_i
\]

Meaning the stability operator can no longer maintain the basin.

The system then invokes:

\[
S_j(s)
\]

where \(j\) is the axis whose basin contains the new state.

All hand‑offs have been validated.

---

# ⭐ 5. **Threshold Safety Conditions**

To prevent runaway activation:

- No axis may activate if another axis is already stabilizing the same domain.  
- Axis 05 overrides all conflicts.  
- Axis 09 cannot override Axis 01.  
- Axis 06 cannot override Axis 02.  
- Axis 08 cannot override Axis 03.  
- Axis 07 cannot override Axis 04.

This ensures **no basin collapse** and **no axis dominance**.
