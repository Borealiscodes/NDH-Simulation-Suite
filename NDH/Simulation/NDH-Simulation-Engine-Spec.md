# 📘 **NDH Simulation Engine Spec**  
### *A Low‑Tech‑Capable, Multi‑Logic, Manifold‑Safe Runtime Engine*

---

## 🌱 **1. Engine Purpose**
The NDH Simulation Engine is designed to:

- generate and host **dimensional experiences**  
- operate in **low‑tech or offline environments**  
- maintain **stability**, **coherence**, and **containment**  
- run on **minimal cognitive or computational resources**  
- support **branching adventures** like 1D–43D  
- integrate with NDH architecture via Integration Guide

This engine is the **runtime**, not the content.

---

## 🧱 **2. Engine Architecture Overview**
The NDH Simulation Engine consists of four core subsystems:

- **Governance Spine** — stability, invariants, safety  
- **Logic Router** — epistemic, mythic, aesthetic, symbolic routing  
- **Manifold Generator** — dimensional geometry  
- **Runtime Core** — execution, synthesis, traversal

These subsystems form a **closed loop**:

```
Governance → Logic → Manifold → Runtime → Governance
```

## 🧭 **3. Engine Operating Modes**
The engine supports three operating modes:

- **Mode 1: Static Simulation**  
  Paper‑portable, offline, human‑portable.  
  No compute required.

- **Mode 2: Guided Simulation**  
  Low‑compute devices (basic phone, offline laptop).  
  NDH logic routing + simple state machine.

- **Mode 3: Dynamic Simulation**  
  Full NDH manifold traversal.  
  Multi‑logic routing + narrative physics.

All modes share the same **core engine**, scaled by environment.

## 🧠 **4. Logic Routing System**
The engine uses NDH’s four logic types:

- **Epistemic Logic** — structure, rules, clarity  
- **Mythic Logic** — archetypes, symbolic intensities  
- **Aesthetic Logic** — tone, vibe, coloration  
- **Symbolic Logic** — motifs, patterns, anchors

Routing rules:

- Mythic → Aesthetic (safe)  
- Mythic → Symbolic (structured)  
- Symbolic ↔ Epistemic (bridge)  
- Mythic → Epistemic (bifurcation only)

This prevents collapse, drift, or overload.
