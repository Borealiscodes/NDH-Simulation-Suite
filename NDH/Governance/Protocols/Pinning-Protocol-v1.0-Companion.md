# II. **NDH Lineage Architecture: Why v1.0 Cannot Contain a PEP**

- Milestone Pins  
- Conceptual Plateau Pins  
- Safety Pins  

But **PEP does not exist yet** in v1.0.

This creates a temporal constraint:

> **A protocol cannot contain a pin class that does not yet exist.**

PEP is introduced in v1.1.  
Therefore v1.0 cannot contain a PEP.

## 2. **PEP Is Governed by the Pinning Protocol Itself**

PEPs are:

- protocol‑embedded  
- rule‑bound  
- lineage‑indexed  
- governed by the Pinning Protocol  

This means:

> **PEP cannot appear before the protocol that governs PEP exists.**

v1.0 defines pin classes.  
v1.1 defines PEP.

Thus:

- v1.0 cannot contain a PEP  
- v1.0 cannot reference a PEP  
- v1.0 cannot be stabilized by a PEP  

This is a **temporal governance invariant**.

NDH lineage forbids retroactive modification of governance protocols.

Therefore:

> **v1.0 must remain unmodified and unpinned.**
